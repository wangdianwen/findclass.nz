# Feature: Trust Badge System - 信任标识系统

> **功能模块**: 搜索与发现 | **优先级**: P0 | **排期**: Phase 1 | **状态**: 待开发

---

## 一、功能概述

### 1.1 功能描述

信任标识系统是平台的核心差异化功能，通过多维度评估为课程数据赋予信任等级和标识，帮助用户识别可靠课程信息。系统采用**五级信任体系**，结合数据来源、验证状态、质量评分等因素综合评定。

### 1.2 核心价值

- **差异化竞争力**: 注册用户数据1.5x权重优先展示，与聚合数据形成差异
- **用户信任建立**: 清晰的信任标识帮助用户做决策
- **数据质量导向**: 激励优质数据供给，提升平台整体质量
- **合规支撑**: 支持GDPR/PIPL数据来源说明要求

### 1.3 用户故事

```
作为 家长
我希望 一眼看出哪些是平台认证的可靠课程
以便 放心选择合适的课程

作为 教师
我希望 发布在平台上的课程能获得信任标识
以便 吸引更多学生

作为 平台运营
我希望 通过信任标识引导数据质量提升
以便 打造高质量的课程数据库
```

## 二、信任等级体系

### 2.1 五级信任等级

| 等级 | 标识 | 名称 | 评分范围 | 搜索权重 | 说明 |
|------|------|------|----------|----------|------|
| **S级** | 🏆⭐ | 平台认证+高质量 | 95-100 | 1.5x × 1.3 = **1.95x** | 平台深度认证，数据完美 |
| **A级** | 🏆 | 平台认证 | 80-94 | 1.5x × 1.2 = **1.8x** | 平台认证，数据优质 |
| **B级** | ✅⭐ | 来源验证+高质量 | 75-89 | 1.0x × 1.1 = **1.1x** | 来源验证，质量良好 |
| **C级** | ✅ | 来源验证 | 60-74 | 1.0x × 1.0 = **1.0x** | 基本验证，质量一般 |
| **D级** | 📱 | 社群来源 | 0-59 | 0.8x × 0.8 = **0.64x** | 未验证，信任度低 |

### 2.2 信任等级计算公式

```
最终评分 = 基础质量分 × 来源权重 × 验证权重 × 新鲜度权重

说明：
- 基础质量分: 0-100，根据数据完整度、准确性等计算
- 来源权重: 注册用户1.5x, Gumtree 1.0x, Facebook 0.8x
- 验证权重: 平台认证1.2x, 来源验证1.0x, 未验证0.5x
- 新鲜度权重: 7天内1.2x, 30天内1.0x, 90天内0.8x, 90天外0.5x
```

### 2.3 信任标识展示规则

| 场景 | 展示内容 | 位置 |
|------|----------|------|
| 课程列表 | 等级标识+名称 | 标题前缀 |
| 课程详情 | 信任等级说明 | 信任信息区块 |
| 搜索结果 | 信任等级图标 | 课程卡片右上角 |
| 教师页 | 认证标识 | 教师名称旁 |

## 三、信任评估维度

### 3.1 评估维度表

| 维度 | 权重 | 评分规则 |
|------|------|----------|
| **数据来源** | 30% | 平台注册用户100, Gumtree 70, Facebook 50 |
| **验证状态** | 25% | 平台认证100, 来源验证60, 未验证30 |
| **信息完整度** | 20% | 完整度100%，按比例计算 |
| **用户反馈** | 15% | 好评率×100，有举报扣分 |
| **时效性** | 10% | 7天内100%，每月递减 |

### 3.2 各维度详细规则

#### 3.2.1 数据来源评分

| 数据来源 | 评分 | 说明 |
|----------|------|------|
| 平台注册用户发布 | 100 | 第一方数据，最高信任 |
| 资质认证教师发布 | 100 | 平台认证教师 |
| Gumtree抓取 | 70 | 有一定可信度 |
| Facebook抓取 | 50 | 社群信息，可信度一般 |
| 其他来源 | 30 | 未知来源，需谨慎 |

#### 3.2.2 验证状态评分

| 验证状态 | 评分 | 说明 |
|----------|------|------|
| 平台深度认证 | 100 | 人工审核，资质验证 |
| 平台基础认证 | 80 | 邮箱/手机验证 |
| 来源验证 | 60 | 数据来源可追溯 |
| 未验证 | 30 | 原始社群信息 |

#### 3.2.3 信息完整度评分

| 字段 | 完整度得分 | 说明 |
|------|------------|------|
| 课程名称 | 15分 | 有标题 |
| 课程描述 | 20分 | 有详细描述 |
| 价格 | 15分 | 有明确价格 |
| 教师信息 | 15分 | 有教师介绍 |
| 上课安排 | 15分 | 有时间地点 |
| 联系方式 | 10分 | 有联系方式 |
| 标签分类 | 10分 | 有分类标签 |

#### 3.2.4 时效性评分

| 时间范围 | 评分 | 说明 |
|----------|------|------|
| 0-7天 | 100% | 最新鲜 |
| 7-30天 | 90% | 较新 |
| 30-90天 | 70% | 一般 |
| 90-180天 | 50% | 较旧 |
| 180天以上 | 30% | 可能过期 |

## 五、数据模型设计

### 5.1 信任等级表

```sql
CREATE TABLE trust_levels (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    course_id       UUID NOT NULL REFERENCES courses(id),
    
    -- 评分维度
    source_score    INTEGER NOT NULL DEFAULT 0,      -- 数据来源评分
    verification_score INTEGER NOT NULL DEFAULT 0,   -- 验证状态评分
    completeness_score INTEGER NOT NULL DEFAULT 0,   -- 信息完整度评分
    feedback_score  INTEGER NOT NULL DEFAULT 0,      -- 用户反馈评分
    freshness_score INTEGER NOT NULL DEFAULT 0,      -- 时效性评分
    
    -- 综合评分
    total_score     INTEGER NOT NULL DEFAULT 0,      -- 综合评分 0-100
    trust_level     VARCHAR(10) NOT NULL,            -- S/A/B/C/D
    
    -- 权重因子
    source_weight   DECIMAL(3,2) NOT NULL DEFAULT 1.0,
    verification_weight DECIMAL(3,2) NOT NULL DEFAULT 1.0,
    freshness_weight DECIMAL(3,2) NOT NULL DEFAULT 1.0,
    
    -- 元数据
    calculated_at   TIMESTAMP DEFAULT NOW(),
    updated_at      TIMESTAMP DEFAULT NOW(),
    
    UNIQUE(course_id)
);

CREATE INDEX idx_trust_levels_course ON trust_levels(course_id);
CREATE INDEX idx_trust_levels_level ON trust_levels(trust_level);
```

### 5.2 信任事件表

```sql
CREATE TABLE trust_events (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    course_id       UUID NOT NULL REFERENCES courses(id),
    user_id         UUID REFERENCES users(id),
    
    event_type      VARCHAR(50) NOT NULL,  -- positive_review, negative_report, verification
    event_value     INTEGER NOT NULL,       -- 评分变化值
    description     TEXT,
    
    created_at      TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_trust_events_course ON trust_events(course_id);
```

## 七、信任更新机制

### 7.1 触发信任更新的事件

| 事件类型 | 触发条件 | 评分影响 |
|----------|----------|----------|
| 新数据 | 新课程发布 | 初始评分计算 |
| 用户举报 | 用户举报课程 | -5到-20分 |
| 正面评价 | 用户给出好评 | +1到+5分 |
| 数据过期 | 超过90天未更新 | -10分 |
| 验证通过 | 通过来源验证 | +10分 |
| 认证升级 | 获得平台认证 | +20分 |

### 7.2 定时任务

| 任务 | 执行频率 | 说明 |
|------|----------|------|
| 信任分重新计算 | 每天凌晨 | 重新计算所有课程的信任分 |
| 过期数据降权 | 每天凌晨 | 降低过期数据的权重 |
| 举报数据审核 | 实时 | 审核用户举报，处理信任分 |

### 8.1 信任标识组件

```tsx
interface TrustBadgeProps {
  level: 'S' | 'A' | 'B' | 'C' | 'D';
  showDetails?: boolean;
  size?: 'small' | 'medium' | 'large';
}

export function TrustBadge({ level, showDetails = false, size = 'medium' }: TrustBadgeProps) {
  const config = {
    S: { icon: '🏆⭐', name: '平台认证+高质量', color: '#52c41a', score: 95 },
    A: { icon: '🏆', name: '平台认证', color: '#1890ff', score: 80 },
    B: { icon: '✅⭐', name: '来源验证+高质量', color: '#722ed1', score: 75 },
    C: { icon: '✅', name: '来源验证', color: '#faad14', score: 60 },
    D: { icon: '📱', name: '社群来源', color: '#d9d9d9', score: 30 },
  };
  
  const item = config[level];
  
  return (
    <div className={`trust-badge trust-badge-${size}`}>
      <span className="trust-icon">{item.icon}</span>
      {showDetails && (
        <div className="trust-details">
          <span className="trust-name">{item.name}</span>
          <span className="trust-score">{item.score}分</span>
        </div>
      )}
    </div>
  );
}
```
