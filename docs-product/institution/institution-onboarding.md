# Feature: Institution Onboarding - 机构入驻流程

> **功能模块**: 机构系统 | **优先级**: P0 | **排期**: Phase 2 | **状态**: 新建

---

## 一、功能概述

### 1.1 功能描述

机构入驻流程是平台付费用户获取的核心功能，允许教育机构申请入驻平台，经过资质审核后获得机构认证，即可自主发布和管理课程、购买套餐、使用推广工具。

### 1.2 核心价值

- **付费用户获取**: 机构付费是平台主要收入来源
- **高质量内容**: 机构认证保证课程质量
- **差异化服务**: 机构专属功能提升粘性
- **生态建设**: 吸引优质机构长期合作

### 1.3 用户故事

```
作为 机构负责人
我希望 申请入驻平台并通过认证
以便 发布机构课程，吸引更多学生

作为 平台运营
我希望 审核机构资质
以便 确保平台上都是正规可靠的教育机构

作为 家长
我希望 看到平台认证的机构
以便 选择可信赖的教育机构
```

### 2.2 入驻类型

| 类型 | 说明 | 所需资料 | 认证等级 | 套餐选择 |
|------|------|----------|----------|----------|
| **小型机构** | 1-5名教师 | 营业执照、法人身份、机构信息 | 🏫 认证 | 基础版/专业版 |
| **中型机构** | 6-20名教师 | 营业执照、法人身份、机构信息、教师资质 | 🏫🏫 高级认证 | 专业版/企业版 |
| **大型机构** | 20+教师 | 营业执照、法人身份、机构信息、教师资质、办学许可 | 🏫🏫🏫 旗舰认证 | 企业版/定制版 |

### 2.3 审核时效

| 审核类型 | 审核时长 | 说明 |
|----------|----------|------|
| **小型机构** | 24-48小时 | 自动审核+人工复核 |
| **中型机构** | 48-72小时 | 人工深度审核 |
| **大型机构** | 3-5天 | 额外背景调查 + 实地考察（可选） |

## 三、资料填写设计

### 3.1 入驻表单字段

| 字段名 | 类型 | 必填 | 说明 |

## 六、审核流程设计

### 6.1 自动审核规则

| 检查项 | 规则 | 不通过处理 |
|--------|------|------------|
| 必填字段 | 所有必填字段已填写 | 阻止提交 |
| 邮箱格式 | 符合邮箱格式 | 提示修改 |
| 手机格式 | 符合新西兰手机格式 | 提示修改 |
| 文件格式 | 符合允许的格式 | 提示修改 |
| 文件大小 | 小于最大限制 | 提示修改 |
| 重复申请 | 同一邮箱/手机未处理申请 | 提示已有申请 |
| 营业执照号码 | 格式校验 | 提示修改 |

### 6.2 人工审核标准

| 审核维度 | 检查内容 | 通过标准 |
|----------|----------|----------|
| 资质真实性 | 证件信息与提交信息一致 | 一致或合理差异 |
| 机构真实性 | 营业执照可查证 | 可查证或合理 |
| 信息完整性 | 介绍信息充实 | 合理充实 |
| 形象展示 | Logo、照片符合规范 | 符合专业标准 |
| 历史记录 | 无严重违规历史 | 无 |

### 6.3 审核结果通知

| 结果类型 | 通知方式 | 通知内容 |
|----------|----------|----------|
| 审核通过 | 邮件+站内信 | 恭喜通过，可选择套餐并支付 |
| 审核拒绝 | 邮件+站内信 | 拒绝原因，可重新申请 |
| 需补充材料 | 邮件+站内信 | 需要补充的具体材料 |

| 风险 | 可能性 | 影响 | 应对措施 |

| 虚假申请 | 中 | 中 | 资质验证，人工审核，第三方查证 |

| 支付失败 | 低 | 中 | 多种支付方式，客服支持 |
| 套餐选择困难 | 中 | 低 | 推荐套餐，突出性价比 |
