# Feature: Admin Backend System - 管理员后台系统

> **功能模块**: 管理员系统 | **优先级**: P0 | **排期**: Phase 1-2 | **状态**: 新建

---

## 一、功能概述

### 1.1 功能描述

管理员后台是平台运营的核心系统，提供用户管理、机构管理、课程审核、内容管理、数据统计等完整功能，支持平台日常运营。

### 1.2 核心价值

- **运营效率**: 集中管理，提升运营效率
- **质量控制**: 审核机制，保证内容质量
- **数据决策**: 数据统计，支持运营决策
- **安全保障**: 权限管理，保护系统安全

### 1.3 用户角色

| 角色 | 说明 | 权限范围 |
|------|------|----------|
| **超级管理员** | 系统最高权限 | 所有功能 |
| **运营管理员** | 日常运营 | 用户管理、机构管理、课程审核 |
| **内容管理员** | 内容管理 | 内容审核、反馈处理 |
| **财务管理员** | 财务管理 | 收入统计、退款处理 |
| **客服管理员** | 客服支持 | 用户反馈、申诉处理 |

## 四、API 设计

### 4.1 API 列表

| 方法 | 路径 | 功能 | 说明 |
|------|------|------|------|
| **用户管理** |
| GET | /api/v1/admin/users | 获取用户列表 | 分页、筛选 |
| GET | /api/v1/admin/users/:id | 获取用户详情 | 完整信息 |
| PUT | /api/v1/admin/users/:id | 更新用户 | 状态、权限 |
| PUT | /api/v1/admin/users/:id/disable | 禁用用户 | |
| POST | /api/v1/admin/users/:id/notify | 发送通知 | |
| **机构管理** |
| GET | /api/v1/admin/institutions | 获取机构列表 | 分页、筛选 |
| GET | /api/v1/admin/institutions/:id | 获取机构详情 | 完整信息 |
| PUT | /api/v1/admin/institutions/:id/status | 更新状态 | 正常/暂停/冻结 |
| POST | /api/v1/admin/institutions/:id/subscription | 修改套餐 | |
| **入驻审核** |
| GET | /api/v1/admin/applications | 获取申请列表 | 待审核优先 |
| GET | /api/v1/admin/applications/:id | 获取申请详情 | 完整信息 |
| PUT | /api/v1/admin/applications/:id/review | 审核申请 | 通过/拒绝/补充 |
| **课程管理** |
| GET | /api/v1/admin/courses | 获取课程列表 | 分页、筛选 |
| GET | /api/v1/admin/courses/:id | 获取课程详情 | 完整信息 |
| PUT | /api/v1/admin/courses/:id/status | 更新状态 | 上架/下架/审核 |
| **财务管理** |
| GET | /api/v1/admin/finance/revenue | 收入统计 | 分时间、类型 |

### 6.2 权限验证

```typescript
// 权限验证中间件
async function verifyPermission(
  adminId: string,
  resource: string,
  action: string
): Promise<boolean> {
  const admin = await getAdmin(adminId);
  
  // 超级管理员拥有所有权限
  if (admin.role === 'super_admin') {
    return true;
  }
  
  // 检查权限配置
  const permission = `${resource}:${action}`;
  return admin.permissions.includes(permission);
}
```

### 14.3 评论处理规则

| 场景 | 处理方式 | 触发条件 |
|------|----------|----------|
| **正常评论** | 保留显示 | 无举报、内容合规 |
| **举报待审** | 人工审核 | 被举报后进入待审 |
| **违规评论** | 删除隐藏 | 涉黄、暴力、恶意攻击 |
| **虚假评论** | 删除警告 | 刷好评/差评、机器人 |
| **争议评论** | 保留+标记 | 双方各执一词，保留展示 |

## 十五、封禁与权限管理

### 15.1 封禁类型

| 类型 | 范围 | 时长 | 触发条件 |
|------|------|------|----------|
| **临时封禁** | 部分功能 | 7/30/90天 | 轻微违规 |
| **永久封禁** | 全部功能 | 永久 | 严重违规 |
| **机构停用** | 机构功能 | 可恢复 | 欠费/违规 |
| **课程下架** | 课程展示 | 可恢复 | 内容问题 |

### 15.2 封禁原因分类

| 类别 | 说明 | 示例 |
|------|------|------|
| **违规内容** | 发布违法、违规内容 | 色情、暴力、诈骗 |
| **虚假信息** | 提供虚假信息 | 虚假资质、虚假评价 |
| **恶意行为** | 恶意刷单、攻击 | 刷好评、恶意投诉 |
| **诈骗行为** | 涉嫌诈骗 | 收款不服务 |
| **投诉过多** | 被多次投诉 | 退款、差评 |
